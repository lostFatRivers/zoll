Когда Вы используете Hudson для автоматизации сборок, иногда удобно иметь возможность параметризировать сборку, т.е. запросить пользователя заполнить набор параметров, которые будут доступены во время процесса сборки. Например, Вы можете создать запускаемый по требованию проект для тестирования, где пользователь может загрузить бинарный zip файл для тестирования.

Эта секция конфигурирует какие параметры Ваш проект будет принимать. Параметры выделяются по их именам, таким образом Вы можете использовать множество параметров с уникальными именами.

Для получения деталей по этой функции используйте Wiki раздел.

Для этого свойства возможно использовать Проектное каскадирование. Детали доступны в этом документе.
